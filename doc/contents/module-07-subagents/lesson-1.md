# サブエージェントの基本

## はじめに

Claude Codeには「サブエージェント」と呼ばれる強力な機能があります。これは、Claude Code自身が別のClaudeエージェントを起動して、タスクを委任できる仕組みです。このレッスンでは、サブエージェントとは何か、どのように動作するか、そして親エージェントと子エージェントの関係について学びます。

## サブエージェントとは

サブエージェントとは、メインのClaude Codeセッション（親エージェント）が必要に応じて起動する、独立したClaude Codeインスタンスのことです。

### 基本的な概念

```
親エージェント（メインセッション）
    ├── サブエージェント1（検索タスク担当）
    ├── サブエージェント2（コード分析担当）
    └── サブエージェント3（テスト生成担当）
```

親エージェントは複数のサブエージェントを同時に起動し、それぞれに独立したタスクを割り当てることができます。サブエージェントはタスクを完了すると結果を親エージェントに返し、親エージェントがその結果を統合します。

### なぜサブエージェントが必要か

大規模なコードベースや複雑なタスクに取り組む際、単一のエージェントには以下のような制約があります。

- コンテキストウィンドウの制限（一度に処理できる情報量の上限）
- 直列処理による時間的コスト
- 一つのエージェントに全ての責務を負わせることで生じる複雑さ

サブエージェントを活用することで、これらの課題を克服できます。

## Agentツールの仕組み

Claude CodeがサブエージェントをスポーンするためのメカニズムがAgentツールです。親エージェントは内部的にAgentツールを呼び出すことで、新しいエージェントインスタンスを起動します。

### Agentツールの呼び出し構造

親エージェントがAgentツールを使うとき、以下の情報をサブエージェントに渡します。

```
Agentツール呼び出し:
- prompt: サブエージェントへの指示（何を調べるか、何を行うか）
- tools: サブエージェントが使用できるツールのリスト
```

サブエージェントは、与えられたツール（ファイル読み込み、コード検索、コマンド実行など）を使ってタスクを遂行し、最終的な結果を親エージェントに返します。

### Claude Codeがサブエージェントを自動起動するケース

Claude Codeは以下のような状況でサブエージェントを自動的に起動します。

**1. 広範な検索タスク**

```bash
# ユーザーの指示
> コードベース全体でdeprecatedなAPIの使用箇所をすべて見つけてください

# 内部動作
# 親エージェントがサブエージェントを起動し、
# ディレクトリ単位で並列検索を実行する
```

**2. 複数ファイルの独立した分析**

```bash
# ユーザーの指示
> src/components/以下の全コンポーネントのアクセシビリティをチェックしてください

# 内部動作
# 各コンポーネントファイルを個別のサブエージェントが担当し、
# 並列でチェックを実行する
```

**3. 相互に独立した複数タスク**

```bash
# ユーザーの指示
> APIの型定義を生成しつつ、既存テストのカバレッジレポートも作成してください

# 内部動作
# タスクAとタスクBを独立したサブエージェントに割り当て、
# 並列で実行する
```

## 親子関係とコンテキスト

サブエージェントの重要な特性は、親エージェントとのコンテキストの扱いにあります。

### コンテキストは共有されない

**これが最も重要なポイントです。** サブエージェントは親エージェントの会話履歴やコンテキストを引き継ぎません。

```
親エージェントのコンテキスト:
- ユーザーとの会話履歴
- 過去に読み込んだファイルの内容
- 実行したコマンドの結果
    |
    | （引き継がれない）
    |
    v
サブエージェントのコンテキスト:
- Agentツールで渡されたpromptのみ
- 独自のツール使用による収集情報
```

つまり、親エージェントがサブエージェントに指示を与える際は、必要な情報をすべてpromptに含める必要があります。

### コンテキスト分離のメリットとデメリット

**メリット**

- 各サブエージェントが独立したコンテキストウィンドウを持つため、大量の情報を処理できる
- 並列実行時に互いの処理が干渉しない
- サブエージェントごとに最適化されたコンテキストを提供できる

**デメリット**

- 親が持つ情報をサブエージェントに渡すには明示的な指示が必要
- サブエージェント間でのリアルタイムな情報共有は不可
- 親エージェントが結果を統合するコストが発生する

### 情報伝達のパターン

```bash
# 親エージェントがサブエージェントに適切な情報を渡す例
# （内部的な動作のイメージ）

# 不十分な指示（コンテキスト不足）
prompt: "このファイルのバグを修正してください"
# → サブエージェントはどのファイルか、どんなバグかわからない

# 十分な指示（コンテキスト付き）
prompt: """
src/api/userService.tsのgetUserById関数にバグがあります。
nullチェックが不足しており、ユーザーが存在しない場合に
TypeError: Cannot read properties of null (reading 'email') が発生します。
このファイルを読み込んで、nullチェックを追加して修正してください。
"""
# → サブエージェントは明確なコンテキストを持って作業できる
```

## サブエージェントのライフサイクル

サブエージェントはタスクを完了すると自動的に終了します。

```
起動 → ツールの使用 → 情報収集 → タスク実行 → 結果返却 → 終了
  │                                                        │
  └────────────────────────────────────────────────────────┘
                    一時的な存在
```

サブエージェントが生成したファイルや実行したコマンドの結果は、ファイルシステムやシェル環境を通じて永続化されますが、サブエージェント自体のメモリ（コンテキスト）は終了とともに消えます。

## サブエージェントが使用できるツール

サブエージェントは親エージェントから許可されたツールのみを使用できます。一般的に使用可能なツールには以下が含まれます。

| ツール | 説明 |
|--------|------|
| Read | ファイルの読み込み |
| Write | ファイルの書き込み |
| Edit | ファイルの編集 |
| Bash | シェルコマンドの実行 |
| Grep | コンテンツ検索 |
| Glob | ファイルパターン検索 |

ただし、Agentツールはサブエージェントからさらにサブエージェントを起動するために使用できますが、無限に再帰することを防ぐため、通常は制限があります。

## まとめ

サブエージェントはClaude Codeが複雑なタスクを効率的にこなすための根幹となる機能です。重要なポイントを整理しましょう。

- サブエージェントは親エージェントが起動する独立したClaudeインスタンス
- Agentツールを通じてサブエージェントが起動される
- 親子間でコンテキストは自動的に共有されない
- サブエージェントはタスク完了後に終了し、結果を親に返す
- 広範な検索、並列分析、独立タスクの処理に自動的に活用される
